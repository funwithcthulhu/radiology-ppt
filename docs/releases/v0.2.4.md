# Radiopaedia Case PowerPoint Builder v0.2.4

Patch release for backend reliability during PowerPoint generation.

## Fixed

- Added one automatic backend restart/retry when the local Node service exits during case preparation, image review, source import, or PowerPoint creation.
- Replaced the generic "backend service exited" failure with an exit-code-aware message that includes recent backend output when a retry still fails.
- Drained backend stdout/stderr through stream close so fast service exits do not drop the useful final error lines.
- Switched the Windows installer to fast non-solid compression so local release builds finish predictably.

## Validation

- `dotnet build csharp/RadiologyPpt.App/RadiologyPpt.App.csproj -c Release`
- `npm test`
