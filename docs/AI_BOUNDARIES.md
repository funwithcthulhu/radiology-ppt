# AI Boundaries

This app is an educational PowerPoint preparation tool. It is not a medical
device, does not diagnose disease, and should not be used for clinical
diagnosis, triage, reporting, or patient-care decisions.

## What Uses Model Output

Ollama image scoring is optional and local. When `Use Ollama image review` is
off, `src/ollama-review.mjs` returns selected images unchanged. When the option
is enabled for preparation, or when `Ollama Score Case` is run in the review
window, the backend looks for a local Ollama vision model and calls:

```text
http://127.0.0.1:11434/api/generate
```

The request sends selected local image bytes plus a short case title/diagnosis
query to the local Ollama server. The expected response is only a numeric
`ollamaScore` and short `ollamaReason`. Those fields may affect selected-image
ordering when scoring is enabled and are written to local prepared-case data or
the generated manifest. They are not treated as a diagnosis, report, or clinical
recommendation.

Image-selection rationale is generated from repository logic in
`src/image-candidates.mjs`: annotations, key-image flags, current slices,
frame proximity, series/view diversity, relevance score, and optional Ollama
score. The rationale explains why the app selected a frame for review; it does
not prove that the image is clinically correct.

Teaching points are generated by deterministic text helpers in
`src/radiopaedia-case-text.mjs` from case text, findings, modality, and the
requested context. They are presentation scaffolding and must be reviewed.

Core Review questions come from the bundled practice bank, imported local
sources, or a custom JSON question bank. Imported-source questions are drafted
from extracted source text and local provenance in `src/core_review`, then
validated against the local schema. The app does not claim official ABR
affiliation, board-exam coverage, or clinical accuracy.

## What Remains User Reviewed

In the desktop workflow, the user reviews prepared cases and images before
export. The review window controls the decisions that matter:

- keep, favorite, or skip a case
- reroll to a different case
- re-pick or manually choose images
- remove weak images
- approve the final set before PowerPoint export

Do not export or present a deck just because a case prepared successfully or an
image received a high score.

## Data And Network Boundaries

Normal case preparation contacts Radiopaedia over HTTPS for search results,
case pages, studies, and images. Random mode also uses Radiopaedia search unless
it can satisfy the request from local indexed cases.

Ollama scoring contacts only the local Ollama endpoint on `127.0.0.1` when the
feature is enabled and a model is available. If no model is found or the local
model times out, the app logs a warning and leaves the images unscored.

Core Review source import reads local files and writes extracted data under:

```text
library\board-review\
```

The app also writes local state under `state\`, `cache\`, `scratch\`,
`outputs\`, and `review-sessions\`. These folders are ignored by Git and should
be treated as private local data. Do not import private patient data or source
material you do not have the right to use.

## Failure Modes

Ignore Ollama scores and image-selection rationale when:

- the score is missing because no local model was available or scoring timed out
- the short reason is generic, anatomically wrong, or inconsistent with the image
- the selected image is low quality, cropped poorly, or does not show the needed finding
- the app selected a caption-adjacent PDF crop that does not match the source text
- imported source text was noisy, incomplete, or extracted from the wrong page
- the case source, imported material, or rights-to-use status is uncertain

When in doubt, use the review controls to skip, reroll, re-pick images, or remove
the case from the export.

## Test Evidence

Boundary-sensitive behavior is covered by focused tests:

- `tests/deck-render.test.mjs` checks that case intro slides hide unsafe
  diagnosis/finding text when possible.
- `tests/radiopaedia-parser.test.mjs` checks age/sex intro formatting and
  random-history avoidance.
- `tests/app-store.test.mjs` checks skipped/rejected case and image-decision
  storage used by later random/re-pick flows.
- `tests/image-candidates.test.mjs` checks deterministic image-selection
  rationale from fixture data.
- `tests/request-parser.test.mjs` checks that PDF/binary-looking content is
  rejected by the Cases request-list importer.
